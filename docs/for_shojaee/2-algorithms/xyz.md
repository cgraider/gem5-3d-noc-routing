# 🧭 الگوریتم XYZ (برای شجاعی)

شجاعی جان، با ساده‌ترین الگوریتم شروع می‌کنیم تا گرم بشی 🙂. اگر این را خوب بفهمی، بقیه راحت‌تر
می‌شوند چون همه‌شان روی همین مفاهیم سوارند.

**XYZ** = مسیریابیِ **قطعی (deterministic)** سه‌بعدی. نه عاملی، نه یادگیری‌ای، نه حالتی — فقط
یک قانونِ ثابت. شماره‌ی الگوریتم **۴**، توپولوژیِ `Mesh_3D`.

> [!TIP]
> 🟢 XYZ همان **خطِ مبنا (baseline)** است که الگوریتم‌های باهوش‌تر را با آن مقایسه می‌کنیم.
> ساده ولی همیشه درست کار می‌کند.

---

## 💡 ایده‌ی اصلی

همیشه ابعاد را به یک **ترتیبِ ثابت** حل کن: اول X، بعد Y، بعد Z.

یعنی بسته اول در راستای X حرکت می‌کند تا `dest_x == my_x` شود؛ بعد در راستای Y تا هم‌تراز شود؛
بعد در راستای Z. چون **همه‌ی** بسته‌ها همین ترتیب را رعایت می‌کنند، شبکه **بدونِ بن‌بست
(deadlock-free)** می‌ماند، بدون اینکه به VC اضافه نیاز باشد.

> [!IMPORTANT]
> 🟣 نقطه‌ضعفِ XYZ: **شلوغی را نمی‌بیند.** هر مسیرِ همیشگی‌اش را می‌رود، فرقی نمی‌کند آن مسیر
> شلوغ باشد یا خلوت. دقیقاً همین ضعف، همان فضایی است که CAQR و DeepNR می‌خواهند بهترش کنند.

## 🔬 خواندنِ کد

فایل: `src/mem/ruby/network/garnet/RoutingUnit.cc` (حدودِ خط ۳۲۳ تا ۳۶۴).

### قدم ۱: از شماره‌ی روتر، مختصات را در بیاور

```cpp
int plane = num_rows * num_cols;   // تعداد روترهای یک لایه
int my_z = my_id / plane;
int my_y = (my_id % plane) / num_cols;
int my_x = (my_id % plane) % num_cols;
// همین کار برای dest_id → dest_x, dest_y, dest_z
```

این همان فرمولِ آشنای پروژه است (بخشِ ۴.۲ مبانی را یادت هست؟).

### قدم ۲: تصمیمِ ترتیب‌دار — این قلبِ XYZ است

```cpp
if      (dest_x > my_x) outport_dirn = "East";
else if (dest_x < my_x) outport_dirn = "West";
else if (dest_y > my_y) outport_dirn = "South";
else if (dest_y < my_y) outport_dirn = "North";
else if (dest_z > my_z) outport_dirn = "Up";    // +Z با TSV
else if (dest_z < my_z) outport_dirn = "Down";  // -Z با TSV
else  panic("XYZ: مبدأ == مقصد ولی outportCompute صدا زده شد");
```

> 🎓 **آموزش کوچک C++ — زنجیره‌ی `if / else if`**
> این `if / else if` پشتِ‌سرِ‌هم، **خودش همان «ترتیبِ ابعاد» است**. چون به‌محضِ اینکه یک شرط
> درست شود، بقیه نادیده گرفته می‌شوند:
> - اول X چک می‌شود؛ تا وقتی X هم‌تراز نشده، اصلاً نوبت به Y نمی‌رسد.
> - وقتی X هم‌تراز شد (`dest_x == my_x`)، آن دو شرطِ اول رد می‌شوند و می‌رسیم به Y.
> - وقتی X و Y هر دو هم‌تراز شدند، می‌رسیم به Z.
>
> `panic(...)` هم یعنی «این حالت نباید هیچ‌وقت پیش بیاید؛ اگر آمد، با خطا متوقف شو». اینجا
> یعنی بسته‌ای که سرِ مقصد است نباید اصلاً به اینجا برسد.

> [!NOTE]
> 🔵 دقت کن: `dest_y > my_y` یعنی `"South"` (Y رو به پایین زیاد می‌شود) و `dest_y < my_y` یعنی
> `"North"`. این قرارداد را همه‌ی الگوریتم‌ها و فایلِ `Mesh_3D.py` رعایت می‌کنند.

### قدم ۳: جهت را به شماره‌ی پورت تبدیل کن و برگردان

```cpp
assert(m_outports_dirn2idx.count(outport_dirn) > 0);
return m_outports_dirn2idx[outport_dirn];
```

> 🎓 **آموزش کوچک C++ — `assert`**
> `assert(شرط)` یعنی «من **مطمئنم** این شرط درست است؛ اگر نبود، برنامه را همین‌جا با خطا
> بِبَند تا باگ را زود پیدا کنم». اینجا مطمئنیم پورتِ موردنظر واقعاً وجود دارد (مثلاً پورتِ
> `Up` روی روترِ لایه‌ی آخر وجود ندارد — و ترتیب‌داری تضمین می‌کند هیچ‌وقت چنین چیزی نخواهیم).

## 🤔 چرا XYZ خطِ مبناست؟

- **درست و کامل:** هر بسته را روی کوتاه‌ترین مسیرِ ترتیب‌دار می‌رساند (`avg_hops ≈ فاصله‌ی
  منهتن`). نرخِ تحویلش در بارِ متوسط حدودِ ۱۰۰٪ است.
- **بدونِ بن‌بست** — برخلافِ XYِ ساده که روی شبکه‌ی سه‌بعدی کرش می‌کند (چون جهتِ Up/Down را بلد
  نیست).
- **کور نسبت به شلوغی** — زیرِ بارِ سنگین یا ترافیکِ بدخیم (مثلِ `transpose`) نمی‌تواند دورِ
  نقاطِ داغ بپیچد. همین، همان فضایی است که الگوریتم‌های یادگیرنده رویش کار می‌کنند.

## ▶️ اجرا

```bash
./build/ALL/gem5.opt configs/example/garnet_synth_traffic.py \
    --network=garnet --num-cpus=32 --num-dirs=32 \
    --topology=Mesh_3D --mesh-rows=4 --mesh-layers=2 --vcs-per-vnet=2 \
    --routing-algorithm=4 --link-latency=1 --router-latency=1 \
    --sim-cycles=1000000 --synthetic=uniform_random --injectionrate=0.1
```

هیچ ترمینالِ عاملی لازم نیست. برای جزئیاتِ اجرا [اینجا](../3-howto/run.md) را ببین.

➡️ قدمِ بعدی: [CAQR](caqr.md) — اولین الگوریتمِ یادگیرنده.
