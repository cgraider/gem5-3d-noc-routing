# ⭐ روشِ پیشنهادی / Proposed (برای شجاعی)

شجاعی، این آخرین و پیشرفته‌ترین الگوریتم است — همان «روشِ پیشنهادیِ» پروژه که قرار است از همه
بهتر کار کند. خبرِ خوب: **۹۰٪ آن مثلِ DeepNR3D است**، فقط چند چیز را بهتر کرده. پس اگر صفحه‌ی
[DeepNR3D](deepnr3d.md) را خوانده باشی، این یکی خیلی سریع جا می‌افتد 🙂.

**Proposed** = DQNِ بهبودیافته برای شبکه‌ی سه‌بعدی، با **بردارِ حالتِ غنی‌ترِ ۱۰-ویژگی‌ای**.
شماره‌ی الگوریتم **۳**، توپولوژیِ `Mesh_3D`، پورتِ ZMQ برابرِ **۵۵۵۶**.

## 🆚 فرق‌هایش با DeepNR3D (جدولِ کلیدی)

| جنبه | DeepNR3D | **Proposed** |
|---|---|---|
| پورتِ ZMQ | ۵۵۵۵ | **۵۵۵۶** |
| اندازه‌ی حالت | `2N + 8` (۵ ویژگی) | **`2N + 28` (۱۰ ویژگی)** |
| تابعِ خطا | MSE | **Huber (نرم‌ترِ L1)** |
| نرخِ یادگیری | ۰٫۰۱ | **۰٫۰۰۰۳** |

> مقاوم‌تر در برابرِ پاداش‌های پرت.

## 🅰️ سمتِ C++ — تابع `outportComputeProposed()`

(حدودِ خطِ ۱۰۱۹ به بعد در `RoutingUnit.cc`)

ساختارِ کلی **عینِ** DeepNR3D است (ساختِ سوکتِ ZMQ، گرفتنِ پاداش، ساختِ حالت، ماسک، فرستادن/
گرفتن، اعتبارسنجی). فقط دو چیزِ مهم اضافه شده:

### ۱) حافظه‌ی EMA برای شلوغی — مهم‌ترین افزوده

```cpp
static std::map<int, std::array<float, 6>> ema_occ; // به‌ازای هر روتر، ۶ جهت
static const float EMA_ALPHA = 0.1f;

// در هر تصمیمِ این روتر، میانگین را آپدیت کن:
ema_occ[my_id][d] = EMA_ALPHA * buf_occ[d] + (1.0f - EMA_ALPHA) * ema_occ[my_id][d];
```

> 🎓 **آموزش کوچک — EMA یعنی چه؟**
> **EMA** = «میانگینِ متحرکِ نمایی». یعنی یک میانگینِ نرم که **تاریخچه** را هم در نظر می‌گیرد،
> نه فقط لحظه‌ی فعلی. فرمولش: `جدید = α·مقدارِ‌حالا + (1−α)·میانگینِ‌قبلی`. با `α=0.1`، هر
> اندازه‌گیریِ جدید فقط ۱۰٪ اثر می‌گذارد، پس نتیجه آرام و کم‌نوسان است. این به عامل «حسِ
> تاریخیِ شلوغی» می‌دهد، نه فقط یک عکسِ لحظه‌ای.

> 🎓 **آموزش کوچک C++ — `std::array<float, 6>`**
> برخلافِ `std::vector` که طولش پویاست، `std::array<float, 6>` یک آرایه‌ی **ثابتِ ۶تایی** است
> (چون همیشه ۶ جهت داریم). کمی سریع‌تر و تمیزتر وقتی طول از قبل معلوم است.

### ۲) بردارِ حالتِ ۱۰-ویژگی‌ای (`2N + 28`)

f1 تا f5 **عینِ** DeepNR3D هستند. این‌ها اضافه شده‌اند:

| ویژگی | یعنی چه |
|---|---|
| f6 | زمانِ انتظارِ این بسته در روترِ فعلی (نرمال‌شده) — بسته‌ی معطل سیگنالِ «عجله کن» می‌دهد |
| f7 | شلوغیِ EMA هر جهت (تاریخچه‌ی نرمِ شلوغی) |
| f8 | تأخیرِ پیش‌بینی‌شده‌ی لینک |
| f9 | استفاده‌ی لحظه‌ایِ هر جهت |
| f10 | فاصله‌ی باقی‌مانده‌ی وزن‌دارشده با شلوغی |

> [!NOTE]
> 🔵 یک نکته‌ی صادقانه از کد: در پیاده‌سازیِ فعلی، f8 عملاً برابرِ f7 می‌شود (نرمال‌سازی‌اش
> ساده‌سازی شده). اگر خواستی تأخیرِ واقعیِ لینک را پیش‌بینی کنی، باید مخرجِ نرمال‌سازی را عوض
> کنی. این‌جور نکته‌های «هنوز کامل نیست» در کدِ پژوهشی عادی است — اشکالی ندارد، فقط بدان.

### ۳) نگاشتِ اکشن به روترِ بعدی با جدولِ delta (تمیزتر از DeepNR3D)

```cpp
static const int delta[6][3] = {{0,-1,0},{1,0,0},{0,1,0},{-1,0,0},{0,0,1},{0,0,-1}};
int nx = my_x + delta[action][0];
int ny = my_y + delta[action][1];
int nz = my_z + delta[action][2];
int next_id = nz * layer_size + ny * num_cols + nx;
DeepNR::store_terminal(packet_id, next_id == dest_id);
```

> 🎓 **آموزش کوچک — چرا «جدولِ delta»؟**
> به‌جای یک زنجیره‌ی طولانیِ `if/else` برای ۶ جهت، یک جدولِ کوچک می‌سازیم که هر اکشن
> (۰..۵) را به تغییرِ `(dx, dy, dz)` نگاشت می‌کند. مثلاً اکشنِ ۰ (شمال) = `{0,-1,0}` یعنی
> `y` یکی کم شود. کوتاه‌تر، خواناتر و کم‌اشتباه‌تر. تکنیکِ خوبی برای یاد گرفتن!

> [!NOTE]
> 🔵 جالب: Proposed از همان namespace مشترکِ `DeepNR::` برای ذخیره‌ی پاداش/پایان استفاده
> می‌کند. این بی‌خطر است چون این دو الگوریتم هیچ‌وقت هم‌زمان اجرا نمی‌شوند (پورتشان فرق دارد).

## 🐍 سمتِ پایتون — `proposed_agent.py`

شبکه‌ی عصبی **عینِ** DeepNR3D است (همان ۵ لایه با لایه‌ی فشرده‌سازی). تفاوت‌های مهم:

### تابعِ خطای Huber به‌جای MSE

```python
loss = nn.functional.smooth_l1_loss(current_q.squeeze(), target_q)
```

> 🎓 **آموزش کوچک — MSE در برابرِ Huber**
> - **MSE** (میانگینِ مربعاتِ خطا) خطاهای بزرگ را خیلی جدی می‌گیرد (به‌توانِ ۲). یک پاداشِ
>   پرتِ خیلی بزرگ می‌تواند آموزش را به‌هم بریزد.
> - **Huber (smooth L1)** برای خطاهای کوچک مثلِ MSE است ولی برای خطاهای بزرگ ملایم‌تر
>   (خطی). برای مسیریابیِ NoC که پاداش‌ها خیلی نوسان دارند، این آموزش را **پایدارتر** می‌کند.

### فازِ گرم‌کردن (warm-up)

```python
WARM_UP_PHASE = 1000   # تا ۱۰۰۰ قدم، اصلاً آموزش نده
```

اول بافرِ تجربه با ۱۰۰۰ تجربه‌ی متنوع پر می‌شود، بعد آموزش شروع می‌شود. این از یادگیریِ زودهنگام
و بی‌کیفیت جلوگیری می‌کند.

> [!IMPORTANT]
> 🟣 جمع‌بندیِ ذهنی: **Proposed = DeepNR3D + حالتِ غنی‌تر (با حسِ تاریخیِ شلوغی) + آموزشِ
> پایدارتر.** همین. اگر DeepNR3D را فهمیده باشی، این فقط چند پیچِ تنظیم‌شده‌ی بهتر است.
