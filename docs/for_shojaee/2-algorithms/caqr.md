# 🧠 الگوریتم CAQR (برای شجاعی)

آفرین که تا اینجا رسیدی شجاعی! 🎉 حالا اولین الگوریتمِ **یادگیرنده** را می‌بینیم. خبرِ خوب: این
یکی هنوز کاملاً داخلِ C++ است و عاملِ پایتون نمی‌خواهد، پس راحت می‌شود کلش را خواند.

**CAQR** = «مسیریابیِ Q آگاه‌از‌شلوغی» (Congestion-Aware Q-Routing). یک **یادگیریِ تقویتیِ
جدولی (tabular)** روی شبکه‌ی **دوبعدی**. شماره‌ی الگوریتم **۵**، توپولوژیِ `Mesh_XY`.

> [!NOTE]
> 🔵 منبعِ مقاله: Srivastava و همکاران، ۲۰۲۴. CAQR پلِ خوبی است بینِ XYZِ ساده و روش‌های
> عمیق: **یاد می‌گیرد**، ولی یادگیری‌اش آن‌قدر ساده است که در یک صفحه کاملش را می‌فهمی.

---

## 💡 ایده‌ی اصلی

یک **جدولِ Q** نگه می‌داریم که به این سؤال جواب می‌دهد:

> «از روترِ `x`، با مقصدِ `d`، اگر از جهتِ `dir` بروم چقدر هزینه دارد؟»

هرچه **Q کمتر** = آن جهت سریع‌تر/خلوت‌تر است. روتر همیشه جهتی با **کمترین Q** را انتخاب می‌کند.
بعد از هر پرش، Qِ روترِ **قبلی** را با تأخیرِ صفی که واقعاً دیده به‌روز می‌کند — این‌طوری جدول
کم‌کم شلوغیِ واقعی را یاد می‌گیرد و ترافیک دورِ نقاطِ داغ می‌پیچد.

---

## 🗃️ جدولِ Q و پارامترها

```cpp
namespace CAQR {
// qtable[router_id][dest_router][outport_dirn] = مقدارِ Q
static std::map<int, std::map<int, std::map<std::string, double>>> qtable;

static int packet_count = 0;           // تعدادِ کلِ تصمیم‌ها تا حالا
static const int    TRAIN_STEPS = 50;  // طولِ فازِ اکتشاف
static const double ALPHA = 0.5;       // نرخِ یادگیری α
static const double GAMMA = 0.7;       // نرخِ تخفیف γ
static const double EPSILON = 0.5;     // احتمالِ اکتشافِ تصادفی ε
}
```

> 🎓 **آموزش کوچک C++ — `static` و `std::map`های تو‌در‌تو**
> - کلمه‌ی **`static`** اینجا یعنی این متغیر **بینِ صداهای مختلفِ تابع زنده می‌ماند** و
>   مشترکِ همه‌ی روترهاست. اگر `static` نبود، هر بار تابع که صدا زده می‌شد، جدول از نو خالی
>   می‌شد و هیچ‌چیز یاد نمی‌گرفتیم! این کلمه‌ی کلیدی قلبِ «حافظه» در این کد است.
> - **`std::map<int, std::map<int, std::map<string, double>>>`** یعنی یک دیکشنریِ سه‌طبقه:
>   با `router` می‌رسی به یک دیکشنریِ `dest`، با `dest` به یک دیکشنریِ `direction`، و آخرش
>   عددِ `double` (مقدارِ Q). یعنی `qtable[5][20]["East"]` = هزینه‌ی رفتن از روترِ ۵ به‌سمتِ
>   مقصدِ ۲۰ از جهتِ شرق. خانه‌هایی که هنوز ندیدیم، پیش‌فرض `0.0`.

## 🔬 خواندنِ تابع `outportComputeCAQR()`

(حدودِ خطِ ۴۰۴ تا ۴۸۵)

### قدم ۱: جهت‌های «شدنی» را جمع کن

```cpp
std::vector<std::string> feasible;
if (dest_x > my_x) feasible.push_back("East");
if (dest_x < my_x) feasible.push_back("West");
if (dest_y > my_y) feasible.push_back("North");
if (dest_y < my_y) feasible.push_back("South");
```

فقط جهت‌هایی که **فاصله تا مقصد را کم می‌کنند** قبول‌اند (حداکثر ۲ تا در شبکه‌ی دوبعدی).
این‌طوری مسیر همیشه مینیمال می‌ماند (بدونِ دور زدنِ عقب). Q تصمیم می‌گیرد بینِ این یکی‌دو
جهت کدام را برویم.

> 🎓 **آموزش کوچک C++ — `std::vector`**
> `std::vector<T>` یک آرایه‌ی پویاست (طولش کم/زیاد می‌شود). `push_back(x)` یعنی «`x` را به
> تهِ لیست اضافه کن». اینجا داریم لیستِ جهت‌های مجاز را می‌سازیم.

### قدم ۲: انتخابِ جهت — اول اکتشاف، بعد بهره‌برداری

```cpp
bool exploring = (CAQR::packet_count < CAQR::TRAIN_STEPS) &&
                 ((double)rand() / RAND_MAX < CAQR::EPSILON);

if (exploring || feasible.size() == 1) {
    chosen_dirn = feasible[rand() % feasible.size()];   // تصادفی
} else {
    // حریصانه: جهتی با کمترین Q را بردار
    chosen_dirn  = feasible[0];
    double min_q = CAQR::getQ(my_id, dest_id, feasible[0]);
    for (size_t i = 1; i < feasible.size(); ++i) {
        double q = CAQR::getQ(my_id, dest_id, feasible[i]);
        if (q < min_q) { min_q = q; chosen_dirn = feasible[i]; }
    }
}
```

در ۵۰ تصمیمِ اول، با احتمالِ ۵۰٪ تصادفی انتخاب می‌کند (**اکتشاف** = امتحان کردنِ راه‌های نو).
بعد از آن (و همیشه پس از گرم شدن)، جهتِ با **کمترین Q** را برمی‌دارد (**بهره‌برداری**).

> [!TIP]
> 🟢 این همان معضلِ معروفِ **اکتشاف در برابر بهره‌برداری (exploration vs exploitation)** است:
> اولش باید راه‌های مختلف را امتحان کنی تا یاد بگیری، بعد که یاد گرفتی باید از بهترین راه
> استفاده کنی. خیلی از الگوریتم‌های یادگیری همین منطق را دارند.

### قدم ۳: به‌روزرسانیِ Q (فرمولِ معروفِ مقاله) — 🟣 مهم‌ترین بخش

وقتی بسته به روترِ `y` از روترِ قبلیِ `x` می‌رسد، Qِ مربوط به `x` برای پرشِ `x→y→d` را به‌روز
می‌کنیم:

```cpp
if (inport_dirn != "Local") {
    // ۱) از روی جهتی که فلیت وارد شده، روترِ قبلی x را بازسازی کن
    int prev_id = ...;   // مثلاً اگر از شمال آمده، x یک سطر بالاتر است
    // ۲) جهتی که در x به y می‌رسد = عکسِ جهتِ ورود در y
    std::string out_x_to_y = opposite(inport_dirn);
    // ۳) q_y = عمقِ صفِ خروجی در y (سیگنالِ شلوغی)
    double q_y = m_router->getOutputUnit(chosen_outport)->getOutQueue()->getSize();
    const double delta_xy = 1.0;   // تأخیرِ لینک = ۱ سیکل

    double Q_y_z_d     = CAQR::getQ(my_id,  dest_id, chosen_dirn);   // بهترین هزینه‌ی ادامه از y
    double Q_x_y_d     = CAQR::getQ(prev_id, dest_id, out_x_to_y);   // تخمینِ قدیمی در x
    double Q_x_y_d_new = Q_x_y_d +
        CAQR::ALPHA * (CAQR::GAMMA * Q_y_z_d + q_y + delta_xy - Q_x_y_d);
    CAQR::setQ(prev_id, dest_id, out_x_to_y, Q_x_y_d_new);
}
CAQR::packet_count++;
```

فرمولِ مقاله این است:

```
Q_x(y,d)_new = Q_x(y,d)_old + α · ( γ · Q_y(z,d) + q_y + δ_xy − Q_x(y,d)_old )
```

| نماد | در کد | یعنی چه |
|---|---|---|
| `Q_x(y,d)` | `Q_x_y_d` | تخمینِ هزینه در روترِ قبلی x برای رفتن به‌سمتِ d از طریقِ y |
| `Q_y(z,d)` | `Q_y_z_d` | بهترین هزینه‌ی ادامه از روترِ فعلی y به‌سمتِ d |
| `q_y` | `q_y` | **سیگنالِ شلوغی** — عمقِ صفِ خروجی در y (همان «آگاه‌از‌شلوغی») |
| `δ_xy` | `delta_xy` | تأخیرِ ثابتِ لینک = ۱ سیکل |
| `α`, `γ` | `ALPHA`, `GAMMA` | نرخِ یادگیری ۰٫۵، تخفیف ۰٫۷ |

> [!IMPORTANT]
> 🟣 **چرا یک پرش عقب؟** هزینه‌ی واقعیِ یک پرش را تا وقتی بسته به روترِ بعد نرسد نمی‌دانیم
> (چون باید عمقِ صفِ آن‌جا را ببینیم). برای همین هر روتر، Qِ **روترِ قبلی** را با شلوغیِ
> محلیِ خودش (`q_y`) به‌روز می‌کند. بسته‌هایی که همین‌جا تولید شده‌اند (`inport == "Local"`)
> پرشِ قبلی ندارند، پس این به‌روزرسانی را رد می‌کنند.

> 🎓 **آموزش کوچک — این فرمول دقیقاً چه می‌کند؟**
> ساختارِ `Q_new = Q_old + α · (هدف − Q_old)` یعنی «Qِ قدیمی را کمی به‌سمتِ یک هدفِ جدید
> بکِش». `α` می‌گوید چقدر تند بکِش (۰٫۵ یعنی نصفِ راه). «هدف» اینجا = `γ·(هزینه‌ی ادامه) +
> شلوغیِ فعلی + تأخیرِ لینک`. این دقیقاً همان قاعده‌ی کلاسیکِ **Q-learning** است، فقط با یک
> ترمِ شلوغیِ `q_y` اضافه که اسمش را «آگاه‌از‌شلوغی» کرده.
