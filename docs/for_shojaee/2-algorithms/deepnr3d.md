# 🤖 الگوریتم DeepNR3D (برای شجاعی)

خب شجاعی، حالا رسیدیم به بخشِ هیجان‌انگیز 😎 — یک **شبکه‌ی عصبیِ عمیق** که مسیریابی را یاد
می‌گیرد. نترس، قدم‌به‌قدم می‌رویم.

**DeepNR3D** = مسیریابی با **شبکه‌ی Q عمیق (Deep Q-Network / DQN)** برای شبکه‌ی سه‌بعدی.
شماره‌ی الگوریتم **۲**، توپولوژیِ `Mesh_3D`، پورتِ ZMQ برابرِ **۵۵۵۵**.

> [!NOTE]
> 🔵 تفاوتِ بزرگ با CAQR: اینجا «جدول» نداریم. به‌جایش یک **شبکه‌ی عصبی** داریم که از روی یک
> بردارِ عددی (حالت)، برای هر جهت یک «ارزش» (Q) پیش‌بینی می‌کند. و این شبکه در یک برنامه‌ی
> **پایتونِ** جدا زندگی می‌کند.

---

## 🧩 دو تکه: C++ و پایتون

```
RoutingUnit در gem5 (C++)  ──[حالت به‌صورت JSON]──►  deepnr_agent.py (پایتون)
   مشتریِ ZMQ                                         سرورِ ZMQ، پورت ۵۵۵۵
                          ◄──[اکشن: عددی بینِ ۰..۵]──
```

> 🎓 **آموزش کوچک — ZMQ و الگوی REQ/REP**
> **ZMQ** یک کتابخانه برای پیام‌رسانی بینِ دو برنامه است. اینجا از الگوی **درخواست/پاسخ
> (REQ/REP)** استفاده می‌کنیم: یک طرف **درخواست** می‌فرستد و **منتظر** می‌ماند، طرفِ دیگر
> **پاسخ** می‌دهد. gem5 «مشتری» است (حالت را می‌فرستد، منتظرِ اکشن می‌ماند) و پایتون «سرور».
> چون gem5 منتظر می‌ماند، شبیه‌سازی تا وقتی عامل جواب ندهد جلو نمی‌رود.

## 🅰️ سمتِ C++ — تابع `outportComputeDeepNR3D()`

(از حدودِ خطِ ۴۹۲ به بعد در `RoutingUnit.cc`، داخلِ `#ifdef USE_ZMQ`)

### متغیرهای `static` که فقط یک‌بار ساخته می‌شوند

```cpp
static void *zmq_ctx  = nullptr;   // زمینه‌ی ZMQ
static void *zmq_sock = nullptr;   // سوکت
static bool  zmq_initialized = false;
static int   connection_failure_count = 0;
static int   invalid_action_count = 0;
```

> 🎓 **آموزش کوچک C++ — چرا باز هم `static`؟**
> دوباره همان `static`! سوکتِ ZMQ را فقط **یک‌بار** در اولین صدا زدنِ تابع می‌سازیم و بعد
> همان را استفاده می‌کنیم. اگر هر بار سوکتِ نو می‌ساختیم، فاجعه می‌شد. پس `static` یعنی «این
> را یک‌بار بساز و نگه‌دار».

### کارهایی که هر بار انجام می‌شود (هر پرشِ هر بسته)

```
۱. پاداشِ پرشِ قبلی را بگیر    → DeepNR::get_reward(packet_id)
۲. بردارِ حالت را بساز (طول = 2*num_routers + 8)
۳. ماسکِ اکشن‌های مجاز را بساز
۴. حالت را با JSON بفرست      → zmq_send(...)
۵. اکشن را بگیر               ← zmq_recv(...)
۶. اعتبارسنجی کن              اگر بد بود → پایانِ اپیزود
۷. اکشن (عدد) را به پورت تبدیل کن و برگردان
```

### بردارِ حالت — چه چیزهایی به عامل می‌گوییم؟

طول = `2 * num_routers + 8`. یعنی این ویژگی‌ها:

| ویژگی | اندازه | یعنی چه |
|---|---|---|
| f1 | `num_routers` | روترِ فعلی به‌صورتِ **one-hot** (همه ۰ جز خانه‌ی روترِ خودمان که ۱ است) |
| f2 | `num_routers` | روترِ مقصد به‌صورتِ one-hot |
| f3 | ۱ | تعدادِ پرش‌های طی‌شده (نرمال‌شده) |
| f4 | ۱ | فاصله‌ی منهتنِ سه‌بعدی تا مقصد (نرمال‌شده) |
| f5 | ۶ | جای خالیِ بافرِ ۶ جهت (N, E, S, W, Up, Down) |

> 🎓 **آموزش کوچک — «one-hot» یعنی چه؟**
> یعنی یک بردار که فقط **یک** خانه‌اش ۱ است و بقیه ۰. مثلاً اگر ۴ روتر داشته باشیم و روترِ
> فعلی شماره‌ی ۲ باشد: `[0, 0, 1, 0]`. این روشِ استانداردِ نشان دادنِ «کدام یکی از چند تا»
> به یک شبکه‌ی عصبی است.

### ماسکِ اکشن‌های مجاز (خیلی مهم)

```cpp
std::vector<bool> available_actions(6, true);
if (my_y == 0)              available_actions[0] = false; // شمال نداریم (لبه)
if (my_x == num_cols - 1)   available_actions[1] = false; // شرق نداریم
// ... و اگر بافرِ یک جهت پر باشد هم آن جهت ممنوع می‌شود
```

روترهای لبه‌ای نمی‌توانند به بیرونِ شبکه بروند. این ماسک به عامل می‌گوید کدام جهت‌ها واقعاً
ممکن‌اند، تا اکشنِ غیرممکن انتخاب نکند.

### اعتبارسنجی و پایانِ اپیزود

اگر عامل یک اکشنِ نامعتبر بدهد (بیرونِ ۰..۵ یا جهتی که در ماسک ممنوع است)، کد:
۱. پاداشِ `-10` برای این بسته ذخیره می‌کند،
۲. شمارنده‌ی اکشن‌های نامعتبر را زیاد می‌کند،
۳. اگر نرخِ نامعتبرها از ۳۰٪ بگذرد (بعد از ۱۰۰ اکشن)، شبیه‌سازی را تمام می‌کند.

> [!WARNING]
> 🔴 این `fatal()`/پایان عمدی است، نه باگ! این علامتِ «اپیزود تمام شد» به حلقه‌ی آموزش است
> تا gem5 را برای اپیزودِ بعد دوباره اجرا کند. اگر دیدی gem5 وسطِ آموزش بسته شد، اول همین را
> در نظر بگیر.

## 🐍 سمتِ پایتون — `deepnr_agent.py`

### شبکه‌ی عصبی (کلاسِ DQN)

یک شبکه‌ی ۵-لایه:

```
state_size → 64 (فشرده‌سازی) → ReLU
64 → 256 → ReLU
256 → 128 → ReLU
128 → 64 → ReLU
64 → action_size (۶ خروجی، Q هر جهت)
```

> 🎓 **آموزش کوچک — لایه‌ی «فشرده‌سازی» چه کاری می‌کند؟**
> اندازه‌ی ورودی (state_size) بسته به اندازه‌ی شبکه فرق می‌کند (۷۲ برای ۴×۴×۲، ۲۷۲ برای
> ۸×۸×۴). لایه‌ی اولِ `fc_compress` هر ورودی‌ای را به ۶۴ تبدیل می‌کند، تا بقیه‌ی شبکه همیشه
> ثابت بماند. ترفندِ تمیزی برای پشتیبانی از اندازه‌های مختلفِ شبکه.

### حلقه‌ی اصلیِ سرور

```
هر پیامی که از gem5 رسید:
  ۱. JSON را بخوان: state, packet_id, reward, done, available_actions
  ۲. تجربه‌ی قدمِ قبل را ذخیره کن:  agent.remember(last_state, last_action, reward, state, done)
  ۳. اکشن انتخاب کن (epsilon-greedy + ماسک):  agent.select_action(...)
  ۴. اکشن را برگردان:  send({"action": N})
  ۵. هر چند بسته یک‌بار آموزش بده:  agent.train()
  ۶. هر چند صد بسته مدل را ذخیره کن
```

> 🎓 **آموزش کوچک — `epsilon-greedy` و «بافرِ تجربه»**
> - **epsilon-greedy:** با احتمالِ `epsilon` یک جهتِ تصادفیِ مجاز (اکتشاف)، وگرنه جهتی با
>   بیشترین Q از خروجیِ شبکه (بهره‌برداری). `epsilon` به‌مرور کم می‌شود (از ۰٫۹ به‌سمتِ ۰٫۰۱).
> - **ReplayBuffer (بافرِ تجربه):** یک صف که چهارتایی‌های `(s, a, r, s', done)` را نگه می‌دارد.
>   موقعِ آموزش، یک دسته‌ی تصادفی از آن برمی‌داریم. چرا تصادفی؟ تا تجربه‌های پشتِ‌سرِهم (که خیلی
>   شبیهِ هم‌اند) شبکه را گمراه نکنند.

### آموزش (`train()`) به زبانِ ساده

۱. یک دسته از بافرِ تجربه بردار.
۲. Qِ فعلی را از شبکه بگیر.
۳. Qِ هدف را حساب کن: `reward + gamma * max(Q_target(next_state))` (اگر done نباشد).
۴. خطا = MSE بینِ Qِ فعلی و هدف.
۵. وزن‌ها را با گرادیان به‌روز کن، `epsilon` را کمی کم کن.
۶. هر چند قدم، وزن‌های شبکه‌ی هدف را با شبکه‌ی اصلی هم‌گام کن.

> [!IMPORTANT]
> 🟣 لازم نیست ریاضیِ پشتِ DQN را کامل بلد باشی تا کدِ gem5 را بفهمی. سمتِ C++ فقط **حالت را
> می‌فرستد و پورتِ برگشتی را اعمال می‌کند**. کلِ هوشِ ماجرا در پایتون است.

```bash
# ترمینال ۱ — اول عامل (شبکه‌ی ۴×۴×۲ → ۳۲ روتر → state = 2*32+8 = 72)
python3 deepnr_agent.py --port 5555 --state-size 72 --action-size 6 --fresh

./build/ALL/gem5.opt configs/example/garnet_deepnr_traffic.py \
    --network=garnet --num-cpus=32 --num-dirs=32 \
    --topology=Mesh_3D --mesh-rows=4 --mesh-layers=2 --vcs-per-vnet=2 \
    --routing-algorithm=2 --link-latency=1 --router-latency=1 \
    --sim-cycles=100000 --synthetic=uniform_random --injectionrate=0.1
```
