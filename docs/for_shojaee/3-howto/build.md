# 🔨 چطور gem5 را بسازیم / Build (برای شجاعی)

شجاعی جان، قبل از اجرا باید gem5 را **کامپایل** کنی. این صفحه ساده و عملی است.

> [!WARNING]
> 🔴 **خیلی مهم:** gem5 روی **ویندوز** ساخته یا اجرا نمی‌شود! درایوِ `v:\...` فقط یک
> مَپ (mount) است؛ نه کامپایلری روی ویندوز هست نه پوشه‌ی `build/`. **همه‌ی** دستورهای ساختن و
> اجرا باید روی **سرورِ لینوکس** زده شوند:
> ```bash
> source ~/gem5-env/bin/activate
> cd ~/gem5_new/gem5/gem5
> ```

---

## ۱) محیطِ پایتون و وابستگی‌ها

gem5 برای ساختن از پایتونِ ۳٫۱۰ استفاده می‌کند. یک‌بار محیطِ مجازی بساز:

```bash
python3.10 -m venv ~/gem5-env
source ~/gem5-env/bin/activate
pip install torch pyzmq numpy      # برای اجرای عامل‌ها (نه برای ساختن)
```

> 🎓 **آموزش کوچک — «محیطِ مجازیِ پایتون» چیست؟**
> `venv` یک پایتونِ جداگانه و تمیز برایِ این پروژه می‌سازد تا کتابخانه‌هایش با بقیه‌ی سیستم
> قاطی نشود. `source .../activate` آن را «روشن» می‌کند. هر بار که ترمینالِ نو باز کردی، باید
> دوباره `activate` کنی.

---

## ۲) وابستگی‌های سیستمی (مهم: ZMQ)

| برای چه | بسته (اوبونتو) |
|---|---|
| ساختِ پایه | `build-essential scons python3-dev` |
| Protocol Buffers | `libprotobuf-dev protobuf-compiler` |
| خروجیِ آماری HDF5 | `libhdf5-dev` |
| **ارتباطِ DeepNR (ZMQ)** | **`libzmq3-dev`** ← 🟣 برای الگوریتم‌های ۲ و ۳ لازم است |

چک کن ZMQ پیدا می‌شود یا نه:

```bash
pkg-config --exists libzmq && echo "ZMQ OK"
```

> [!IMPORTANT]
> 🟣 اگر ZMQ نباشد، باز هم کامپایل **موفق** می‌شود، ولی DeepNR3D و Proposed موقعِ اجرا
> `fatal()` می‌زنند. چون کلِ کدِ آن‌ها داخلِ `#ifdef USE_ZMQ` است. پس اگر این دو الگوریتم
> کرش کردند، اول ZMQ را چک کن.

## ۳) دستورِ ساختن

```bash
# نسخه‌ی بهینه (همانی که معمولاً اجرا می‌کنی):
scons build/ALL/gem5.opt -j$(nproc)

# نسخه‌ی دیباگ (کندتر، ولی خطاها واضح‌تر، با stack trace):
scons build/ALL/gem5.debug -j$(nproc)
```

> 🎓 **آموزش کوچک — `scons` و `-j$(nproc)`**
> - **`scons`** ابزارِ ساختِ gem5 است (مثلِ `make`). خودش می‌فهمد چه فایل‌هایی عوض شده‌اند و
>   فقط همان‌ها را دوباره کامپایل می‌کند.
> - **`-j$(nproc)`** یعنی «از همه‌ی هسته‌های CPU استفاده کن» (`nproc` تعدادِ هسته‌ها را
>   می‌دهد). بارِ اول ساختن ۲۰ تا ۶۰ دقیقه طول می‌کشد؛ دفعه‌های بعد (وقتی فقط چند فایل عوض
>   شده) چند ثانیه.

> [!NOTE]
> 🔵 این پروژه از پیکربندیِ `ALL` استفاده می‌کند (همه‌ی معماری‌ها و پروتکل‌ها). بعضی اسکریپت‌های
> قدیمی به `build/Garnet_standalone` یا `build/X86` اشاره می‌کنند؛ آن‌ها را به `build/ALL`
> تغییر بده یا یک symlink بساز.

---

## ۴) بعد از تغییر دادنِ کد

کافی است همان دستورِ `scons` را دوباره بزنی. مثلاً بعد از ویرایشِ `RoutingUnit.cc`:

```bash
scons build/ALL/gem5.opt -j$(nproc)   # معمولاً ۱ فایل را دوباره می‌سازد و لینک می‌کند
```

> [!TIP]
> 🟢 برای تکرارِ سریع موقعِ کار روی کدِ مسیریابی، می‌توانی فقط فایل‌های آبجکتِ garnet را
> بسازی (سریع‌تر از ساختِ کامل). جزئیات در نسخه‌ی انگلیسی:
> [`../../3-howto/build.md`](../../3-howto/build.md).

---

## ۵) چک کردنِ نتیجه‌ی ساختن

```bash
./build/ALL/gem5.opt --version
nm build/ALL/gem5.opt | grep -i zmq | head -5   # باید نمادهای zmq را ببینی
```

➡️ حالا که ساختی، برو سراغ [اجرا و ارزیابی](run.md).
